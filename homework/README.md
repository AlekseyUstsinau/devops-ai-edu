# Final Project Requirements

## 🎯 Project Overview

Welcome to the final project of the DevOps AI Education course! This project is designed to challenge you to leverage AI tools effectively while demonstrating your understanding of DevOps principles and best practices.

## 🤖 Core Requirement: AI-Generated Solution

**All code and infrastructure must be generated using AI assistance.** This project tests your ability to:
- Craft effective prompts
- Guide AI toward production-quality solutions
- Iterate and refine AI-generated outputs
- Apply critical thinking to AI suggestions

## 📝 Mandatory Deliverables

### 1. Prompts Documentation File

Create a file documenting your prompts used throughout the project:
- **File name:** `PROMPTS.md` or `prompts.txt`
- **Minimum requirement:** At least **4 prompts** (one per task minimum)
- **Important:** This file **MUST be created manually** by you - not by AI
- **Content should include:**
  - Clear description of what you asked the AI
  - Context you provided
  - Any iterations or refinements you made

> 💡 **Note:** You're encouraged to document more than 4 prompts if you use multiple iterations to achieve your goals. Quality prompting often requires refinement!

### 2. Working Infrastructure and Application Code

Complete all four tasks as described in:
- `task1.md` - Architecture Design
- `task2.md` - Infrastructure as Code
- `task3.md` - CI/CD Pipeline
- `task4.md` - Security and Optimization

## ✅ Evaluation Criteria

Your project will be evaluated on the following dimensions:

### 1. **Functionality** (Critical)
- ✨ Code must be **fully functional**
- 🟢 CI/CD pipeline must show **green status**
- ✅ `terraform apply` job must complete **successfully**
- 🚀 All components must deploy and run as intended

### 2. **Code Quality**
- 📐 Clean, well-structured code
- 🎯 Following best practices and conventions
- 📚 Proper naming conventions
- 🔧 Appropriate use of modules and reusability
- 💬 Meaningful comments where necessary

### 3. **Infrastructure Design**
- 🏗️ Sound architectural decisions
- 🔒 Security considerations implemented
- 💰 Cost optimization awareness
- 📊 Proper resource organization

### 4. **Prompt Engineering Quality**
- 🎨 Well-crafted, specific prompts
- 🔄 Evidence of iterative refinement
- 🎓 Demonstration of learning progression
- 💭 Strategic thinking in AI interaction

## 🎪 Success Looks Like

A successful submission includes:
- ✅ A working GitHub repository
- ✅ Green CI/CD pipeline
- ✅ Successfully deployed infrastructure
- ✅ Manual `PROMPTS.md` file with your prompt history
- ✅ Clean, production-quality code
- ✅ Proper documentation

## 💡 Tips for Success

1. **Start with clear requirements** - Feed AI with precise context
2. **Iterate on outputs** - First generation is rarely perfect
3. **Test frequently** - Validate AI-generated code works as expected
4. **Refine prompts** - Learn from what works and what doesn't
5. **Think critically** - AI is a tool, you're the engineer

## 🌟 Remember

This project is not just about getting code written by AI - it's about demonstrating that you can **effectively collaborate with AI** to produce **professional-grade, working solutions**. The goal is to show that AI-assisted development, when done right, can match or exceed traditional development quality.
